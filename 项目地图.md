# 项目地图

这个文件是给你看的中文总览。

它不是记忆文件，也不是代码文件。它的作用是让你一打开项目，就能知道：

- 这个项目现在是什么阶段
- 每个文件夹是干嘛的
- 真正的记忆以后放在哪里
- 哪些东西是给 Codex/AI 看的
- 下一步应该做什么

## 这个项目是什么

这个项目要做的是 **AI-native Personal Brain**。

简单说，它不是普通记事本，也不是关键词搜索工具。

目标是：

```text
你随便输入想法
-> AI 理解、润色、拆分
-> 形成长期记忆
-> 以后可以语义检索
-> AI 基于证据回答你之前怎么想的
```

现在还没做到完整第二大脑。

现在完成的是两个地基：

1. 数据库地基
2. Memory Router 引路层

## 当前做到哪一步

当前阶段：

```text
已完成：数据库结构
已完成：Memory Router
未完成：AI 记忆提取
未完成：embedding/RAG 检索
未完成：微信机器人
未完成：自动周整理
```

也就是说，现在不是“大脑已经好了”，而是：

```text
大脑的地基和地图已经搭好
真正的记忆生成层还没开始
```

## 根目录文件说明

### `项目地图.md`

就是你现在看的这个文件。

它是给你看的项目导航页。

### `ARCHITECTURE.md`

这是项目蓝图。

它更偏架构原则，主要告诉我和 Codex：

- 这个项目不能做成普通 CRUD
- 不能只做关键词搜索
- 不能用固定分类代替 AI 判断
- 必须保留原始输入
- 必须有 AI 改写后的 atomic memory
- 回答必须基于证据

你可以把它理解成：

```text
项目宪法
```

### `README.md`

这是当前版本的使用说明。

它告诉你现在有哪些命令，比如：

```powershell
python brain.py init-db
python brain.py build-router
python brain.py stats
```

### `brain.py`

这是命令入口。

你在命令行里运行的命令，都是先进这个文件。

它像一个遥控器，本身不存记忆，只负责把命令交给核心代码。

### `brain_index.json`

这是给 Codex/AI 看的总地图。

以后 Codex 要查记忆时，应该先读它，而不是直接打开完整 SQLite 数据库。

它会告诉 AI：

- 主题索引在哪里
- 记忆清单在哪里
- SQLite 数据库在哪里
- 不要全库读取
- 不要没有证据就瞎总结

## 文件夹说明

## `personal_brain/`

这是核心代码文件夹。

可以理解成：

```text
第二大脑系统的发动机房
```

目前重要文件有：

### `personal_brain/schema.py`

这是数据库地基。

它负责创建真正第二大脑需要的表：

```text
raw_messages              原始输入
memory_extraction_runs    AI 处理记录
memories                  AI 改写后的长期记忆
memory_embeddings         语义检索向量
topics                    AI 动态主题
memory_topics             记忆和主题的关系
entities                  AI 识别的人/事/概念
memory_entities           记忆和实体的关系
legacy_memories           旧 demo 数据
```

最重要的是区分两层：

```text
raw_messages = 原始输入，像录音带
memories = AI 整理后的记忆卡片
```

以后不能再把原始聊天直接塞进 `memories`。

### `personal_brain/router.py`

这是 Memory Router。

它负责生成：

```text
brain_index.json
memory/topics.json
memory/memory_manifest.json
```

它的作用是给 Codex/AI 指路：

```text
先看总地图
再看主题
再看记忆清单
最后才按需读数据库证据
```

它不是 RAG，也不是检索大脑，只是引路人。

### `personal_brain/brain.py`

这是核心调度器。

它把命令入口和具体模块连起来。

例如：

```text
brain.py 命令入口
-> personal_brain/brain.py
-> schema.py 或 router.py
```

### `personal_brain/config.py`

这是配置读取器。

它决定：

- 数据库路径
- memory 文件夹路径
- brain_index.json 路径
- 以后模型 API 配置

### `personal_brain/llm.py`

这是以后接 AI 模型的插座。

现在还不是重点。

以后做 AI 改写、AI 分类、AI rerank、AI 回答时会用它。

## `data/`

这是数据库仓库。

最重要的是：

```text
data/personal_brain.sqlite3
```

以后真正的记忆数据会主要存在这里。

现在数据库里已经有正式地基表，但还没有真正的 AI memory。

里面还有一些测试残留文件，比如旧的 sqlite 文件、cache 文件。它们已经在 `.gitignore` 里，不是正式项目内容。

## `memory/`

这是给 Codex/AI 看的索引区。

它不是源数据库。

现在主要有：

```text
memory/topics.json
memory/memory_manifest.json
```

### `memory/topics.json`

主题索引。

现在只有：

```text
legacy_unprocessed
```

意思是：

```text
旧 demo 数据，还没经过 AI 正式整理
```

以后 AI 记忆提取做好后，这里会出现真正的 AI 动态主题。

### `memory/memory_manifest.json`

记忆清单。

它不会存完整记忆正文，而是告诉 Codex：

- 有哪些候选记忆
- 它们大概是什么
- 属于哪些主题
- 如果要看证据，去 SQLite 哪张表哪一条找

## `scripts/`

这是未来放外部适配脚本的地方。

比如以后接：

- 微信机器人
- webhook
- 自动化脚本

现在里面有一些早期壳子，但目前不是重点。

现阶段重点不是微信，而是先把大脑地基做好。

## 真正的记忆以后在哪里

以后真正的记忆分三层：

### 第一层：原始输入

存在：

```text
raw_messages
```

它保存你最原始的话。

作用是保留证据。

### 第二层：AI 记忆

存在：

```text
memories
```

它保存 AI 改写后的 atomic memory。

比如你原话很口语：

```text
我感觉这个别做成普通搜索啊，要 AI 会理解的那种
```

AI 可能整理成：

```text
用户希望 Personal Brain 以 AI 理解和推理为核心，而不是退化为普通关键词搜索工具。
```

这才是真正的长期记忆。

### 第三层：索引和路由

存在：

```text
brain_index.json
memory/topics.json
memory/memory_manifest.json
```

它们帮助 Codex/AI 快速找到相关记忆，不用每次全库读取。

## 哪些文件不是记忆

这些不是记忆：

```text
ARCHITECTURE.md
README.md
项目地图.md
brain.py
personal_brain/*.py
```

它们是项目说明和代码。

真正的记忆以后主要在：

```text
data/personal_brain.sqlite3
memory/*.json
```

## 为什么不建固定分类文件夹

因为固定文件夹会把项目带偏。

比如如果一开始建：

```text
AI记忆/
Codex/
产品想法/
网络VPN/
```

看起来清楚，但问题是：

一条记忆可能同时属于多个主题。

比如：

```text
我希望 Codex 像技术合伙人一样帮我审查 Personal Brain 架构。
```

它同时属于：

```text
Codex
AI-native Personal Brain
产品架构
协作模式
```

如果用文件夹，只能放一个地方，或者复制很多份。

所以我们现在用：

```text
数据库 + 动态 topics + router
```

这样更像大脑，而不是文件柜。

```text

```


配置样例在：

```text
config.example.json
```

```text
Z.AI GLM-5V-Turbo
```

```text
项目能不能稳定调用真实 AI 模型
```


注意：

```text

```

## AI 记忆提取现在做到哪一步

现在已经有第一版 `ingest` 命令：

```powershell
python brain.py ingest "你的想法"
```

它会做这几件事：

```text
1. 把你的原话存入 raw_messages
2. 调用 GLM-5V-Turbo 理解这句话
3. 让模型输出结构化 JSON
4. 把 AI 改写后的 atomic memory 存入 memories
5. 把 AI 识别出的主题存入 topics
6. 把 AI 识别出的实体存入 entities
7. 更新 Memory Router
```

用人话说：

```text
raw_messages = 你的原话，保留证据
memories = AI 整理后的长期记忆卡片
```

注意：

```text
ingest 还不是检索
ingest 还不做 embedding
ingest 还不回答问题
```

它只是让第二大脑开始“长记忆”。
